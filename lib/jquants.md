## 上場銘柄一覧（/listed/info）

### 説明
各上場銘柄の基本情報一覧を返します。主に銘柄コード、会社名、市場区分、業種などが含まれます。

### 代表的なフィールド

| フィールド名 | 型 | 説明 |
|---|---:|---|
| Date | string (YYYY-MM-DD) | データ取得日／適用日 |
| Code | string | 銘柄コード（例: 13010） |
| CompanyName | string | 会社名（日本語） |
| CompanyNameEnglish | string | 会社名（英語） |
| Sector17Code / Sector17CodeName | string / string | 17分類の業種コードと名称 |
| Sector33Code / Sector33CodeName | string / string | 33分類の業種コードと名称 |
| ScaleCategory | string | 規模カテゴリ（例: TOPIX Small 2） |
| MarketCode / MarketCodeName | string / string | 市場コードと市場名（例: 0111 / プライム） |

## 日次株価（/prices/daily_quotes）

### 説明
銘柄ごとの日次終値・始値・高値・安値・出来高などを返します。調整済み価格（Adjustment*）や売買代金（TurnoverValue）なども含まれる場合があります。

### 代表的なフィールド

| フィールド名 | 型 | 説明 |
|---|---:|---|
| Date | string (YYYY-MM-DD) | 対象日 |
| Code | string | 銘柄コード |
| Open | number | 始値 |
| High | number | 高値 |
| Low | number | 安値 |
| Close | number | 終値 |
| Volume | number | 出来高 |
| TurnoverValue | number | 売買代金（出来高×価格の合計） |
| UpperLimit / LowerLimit | number | 制限値（寄与する場合） |
| AdjustmentFactor | number | 株式分割等を考慮した調整係数 |
| AdjustmentOpen/High/Low/Close | number | 調整済価格 |

## 財務情報（/fins/statements）

### 説明
企業の決算短信や有価証券報告書から得られる財務諸表の主要項目を返します。項目数が非常に多く（売上高、営業利益、EPS、総資産、配当など多数）、四半期別・通期別の予測/実績が含まれます。

### 代表的なフィールド（一部抜粋）

| フィールド名 | 型 | 説明 |
|---|---:|---|
| DisclosedDate / DisclosedTime | string | 開示日／時刻 |
| LocalCode | string | 銘柄コード |
| NetSales | number | 売上高 |
| OperatingProfit | number | 営業利益 |
| OrdinaryProfit | number | 経常利益 |
| Profit | number | 当期純利益 |
| EarningsPerShare | number | 一株当たり利益（EPS） |
| TotalAssets | number | 総資産 |
| Equity | number | 純資産 |
| CashFlowsFromOperatingActivities | number | 営業活動によるキャッシュ・フロー |
| ForecastNetSales / ForecastProfit | number | 予想（次期または四半期） |

## 注意点 / 補足

- **利用制限**: J-Quants の各エンドポイントはプラン（Free/有料）によって利用可否・レスポンス範囲が異なります。Free プラン以上で利用可能なエンドポイントが存在します。
- **レスポンス形式**: 実装内では pandas.DataFrame を JSON に変換して返す場合があります（表形式）。そのため、数値が float で返る／NaN が null になる等の変換挙動に注意してください。
- **認証**: J-Quants を利用するためにはアカウントと API の認証情報が必要です。本リポジトリでは `lib/jquants.py` が認証（idToken 取得）を担当します。
- **データの整合性**: 株価や財務の単位（千円／百万円など）や表記揺れがある場合があります。用途に応じて正規化（単位変換、欠損処理）を行ってください。
- **シングルトンパターン**: 本実装では J-Quants クライアントがシングルトンパターンで実装されており、効率的なリソース管理とトークン管理が行われています。
- **自動トークンリフレッシュ**: 24時間有効期限のトークンが自動的にリフレッシュされ、継続的なAPI利用が可能です。

- **CORS対応**: フロントエンドからのアクセスをサポート
